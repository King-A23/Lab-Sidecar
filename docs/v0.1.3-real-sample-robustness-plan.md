# v0.1.3 Real Sample Robustness Plan

Date: 2026-06-24

## Product Intent

v0.1.3 moves Lab-Sidecar from contract-stable alpha artifacts toward robust
real-ish local sample handling, without expanding beyond the CLI-first,
file-first, local-first product boundary.

The user value is practical: local experiment owners should be able to ingest
small real-ish result directories, collect CSV/JSON metrics with clearer
diagnostics, understand what was skipped or mapped, and generate deterministic
figures, reports, slides, validation records, and packages that are useful for
review.

The main workflow remains:

```text
run / ingest -> collect -> figures -> report -> slides
```

Release and comparison evidence continue to build on that path:

```text
run / ingest -> collect -> figures -> report -> slides -> validate -> package -> package-verify
run / ingest -> collect -> compare --save -> validate-comparison -> package-comparison -> package-verify
```

## Scope

In scope for v0.1.3:

- Small real-ish fixtures that exercise messy local result directories without
  adding bulky repository data.
- CSV/JSON collection diagnostics that explain skipped files, configured source
  matches, field alias resolution, missing mapped fields, include/exclude
  behavior, and unit or group conflicts.
- Deterministic figure defaults that behave predictably on training and
  benchmark-style samples.
- Clear unsupported explicit figure-spec diagnostics.
- Deterministic report and slide output that stays readable on real-ish
  samples and names evidence paths, omissions, and partial data clearly.
- Validation and package evidence that proves the generated artifacts are
  package-ready while keeping raw sources, full logs, SQLite, worker audit
  files, and unrelated workspace files out of packages by default.
- Descriptive-only saved comparison artifacts.

## Non-Goals

v0.1.3 must not add or claim:

- Web UI.
- FastAPI or any HTTP service.
- Hosted service behavior, cloud sync, or multi-tenant service features.
- Remote runner or remote execution.
- General multi-agent framework behavior in the product.
- AI-authored conclusions, AI report writing, or default AI chart
  recommendation.
- Advanced analytics, statistical significance, model superiority, paper
  conclusions, or deployment recommendations.
- DAG scheduler or orchestration framework.
- MCP/V2 product-surface expansion.
- JSONL, TensorBoard, full MLflow tracking-store parsing, animation, video,
  interactive charts, or broad recursive workspace scanning by default.

MCP/V2 code and tests are out of scope unless an existing test failure requires
a compatibility fix. Codex supervisor agents and subagents may coordinate this
repository work, but that does not become Lab-Sidecar product architecture.

## Workstreams

### A. Real-ish Sample Fixtures

Add or extend small fixtures that exercise realistic local result shapes:

- `examples/messy-csv-results` with multiple CSV files, debug/scratch files,
  alias-style columns such as `acc`, `val_accuracy`, and `score_pct`, run
  identity fields such as `seed`, `trial`, or `run_id`, and recoverable missing
  or malformed cases.
- `examples/json-benchmark-results` with JSON metric records across multiple
  algorithms or methods, runtime/accuracy/error fields, and bounded skipped or
  failed-record diagnostics.
- Existing `examples/project-presentation-pack` remains the primary
  course/project presentation fixture and may receive README polish or tiny
  data tweaks only if needed.

Fixture constraints:

- Keep files small and text-based.
- Do not commit `.lab-sidecar/` outputs, generated packages, temporary
  package folders, or large binary artifacts.
- Do not introduce external network dependencies.

### B. Collector Diagnostics And Config Guidance

Improve `collect` behavior for messy inputs while preserving explicit config as
the safe path:

- Record clearer skipped-file reasons for unsupported types, excluded files,
  parse failures, missing fields, or no usable metric rows.
- Record configured include/exclude match evidence.
- Record field alias resolution evidence, including the source column matched
  for each normalized target.
- Make missing mapped fields and unit/group conflicts actionable.
- Keep CLI guidance pointed at `collect --config` rather than guessing.

Non-goals:

- No JSONL, TensorBoard, or MLflow tracking-store support.
- No broad recursive scan of the whole workspace by default.
- No automatic unit conversion.
- No AI mapping.

Candidate files:

- `lab_sidecar/collectors/config.py`
- `lab_sidecar/collectors/scan.py`
- `lab_sidecar/collectors/fields.py`
- `lab_sidecar/collectors/service.py`
- `lab_sidecar/collectors/scenario_summary.py`
- `tests/test_collector_bad_inputs.py`
- `tests/test_collection_summary_contract.py`
- New focused real-sample tests as needed.

### C. Figure Defaults And Unsupported-Spec Diagnostics

Improve deterministic chart behavior on richer training and benchmark data:

- Keep default line/bar/box selection stable.
- Prefer predictable defaults for multi-seed, method, runtime, accuracy, loss,
  and error-style fields.
- Make unsupported explicit spec diagnostics clear and bounded.
- Preserve the deterministic renderer and keep bounded fallback opt-in.

Non-goals:

- No AI chart recommendation.
- No interactive charts.
- No advanced statistics or significance charts.
- No animation or video.

Candidate files:

- `lab_sidecar/figures/specs.py`
- `lab_sidecar/figures/service.py`
- `lab_sidecar/figures/render.py`

### D. Report And Slides Quality On Real-ish Samples

Make deterministic report and slide artifacts more useful for real-ish local
samples:

- Avoid overclaiming results or turning summaries into conclusions.
- Keep slides readable and not crowded.
- Name evidence paths for metrics, figures, reports, and diagnostics.
- Surface skipped inputs, omissions, failed rows, or partial data with cautious
  diagnostic wording.
- Preserve the descriptive-only comparison report boundary.

Non-goals:

- No AI report writing.
- No paper-ready research conclusions.
- No PowerPoint animation.
- No design-system rewrite.

Candidate files:

- `lab_sidecar/reports/service.py`
- `lab_sidecar/reports/templates.py`
- `lab_sidecar/slides/service.py`
- `tests/test_report_summary_contract.py`
