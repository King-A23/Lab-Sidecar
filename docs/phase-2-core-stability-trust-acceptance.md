  `.[dev,mcp]`, `python -m pytest -q`, then `python -m build`.
- `pyproject.toml` baseline: `dev` extra includes `pytest>=8.0`; no configured
  `ruff`, type-check, coverage, or pytest-cov dependency yet.
- Public schema-like artifacts already generated by the product include
  `manifest.json`, `metrics/collection-summary.json`,
  `metrics/scenario-summary.json`, `figures/figure-summary.json`,
  `reports/report-summary.json`, `slides/slides-summary.json`,
  `provenance/traceability.json`, `package-summary.json`, and
  `artifact-index.json`.
- `TaskRecord`, `TaskPaths`, and `ArtifactRecord` are Pydantic models in
  `lab_sidecar/core/models.py`; most summary artifacts are currently assembled
  as dictionaries in their service modules.
- `metrics/scenario-summary.json` has a documented contract and focused tests
  for boundedness, free-text omission, missing primary metrics, parseable mixed
  numeric strings, and descriptive seed aggregates.
- Manual CLI run currently executes user-provided command strings through the
  shell path. `lab_sidecar/runner/service.py` contains `subprocess.Popen(...)`
  with `shell=True` for foreground task execution.
- Background execution starts `lab_sidecar.runner.worker` through
  `subprocess.Popen(...)` and passes the command string through worker args.
- README and SECURITY already state that manual CLI `run` is user-explicit
  local command execution, not an OS sandbox.
- Existing docs and tests distinguish manual CLI `run` from MCP/V2 bounded
  command paths and host-facing preview gates.
