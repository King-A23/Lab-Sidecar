# v0.1.5 Install And Release Trust Acceptance

Date: 2026-06-25

## Scope

This acceptance record covers the v0.1.5 Public Install and Release Trust
slice. It hardens repository release checks, installed wheel verification, and
small public artifact-contract fixtures without expanding Lab-Sidecar beyond
the CLI-first, file-first, local-first workflow.

Primary path preserved:

```text
run / ingest -> collect -> figures -> report -> slides
```

## Implemented Behavior

- Version metadata is advanced to `0.1.5` in `pyproject.toml` and
  `lab_sidecar/__init__.py`.
- `CHANGELOG.md` includes a `0.1.5` entry dated 2026-06-25.
- `scripts/release_check.py` checks release metadata and local dist artifacts
  before a maintainer creates any tag or release.
- The release check reads wheel `METADATA` and sdist `PKG-INFO` so wrong
  contents with right-looking filenames are rejected.
- The release check runs unstaged, staged, and committed `HEAD` whitespace
  checks; `--require-clean-git` additionally enforces an empty porcelain status
  for maintainer pre-tag use.
- `scripts/release_asset_smoke.py` installs a supplied wheel into an isolated
  virtual environment and verifies the installed CLI with a compact local
  workflow. With `--version`, it also verifies the installed `lab-sidecar`
  package metadata version.
- `tests/fixtures/artifact_contract/v1/` contains small sanitized JSON-only
  golden fixtures for `manifest.json`, `metrics/collection-summary.json`,
  `provenance/traceability.json`, `package-summary.json`, and
  `artifact-index.json`.
- README and release checklist docs now distinguish GitHub release artifact
  installation from editable development installs.

## Known Limits

- The new release scripts do not create tags, create GitHub releases, upload
  artifacts, or publish to PyPI.
- `release_asset_smoke.py` can accept HTTP(S) wheel URLs for manual checks, but
  default tests do not depend on network access.
- Golden fixtures are small public-contract examples, not a complete generated
  `.lab-sidecar/` task output snapshot.
- Package and traceability artifacts may still contain local paths, command
  previews, or argv metadata that humans should review before sharing. The
  hardened omission contract is about full logs, raw source bodies, worker
  prompt/response bodies, artifact bytes, SQLite, sandbox files, and unrelated
  workspace files.
- No OS sandboxing, malware detection, hosted service, Web UI, FastAPI/HTTP
  service, remote runner, cloud sync, default AI analysis, statistical
  significance, model-superiority claim, or MCP/V2 product expansion was added.
