# v0.1.6 First-User Onboarding And Recipe Trust Acceptance

Date: 2026-06-25

## Scope

This acceptance record covers the v0.1.6 First-User Onboarding and Recipe Trust
slice. It improves release-wheel onboarding, recipe navigation, and release
asset evidence without expanding Lab-Sidecar beyond the CLI-first, file-first,
local-first workflow.

Primary path preserved:

```text
run / ingest -> collect -> figures -> report -> slides -> validate -> package -> package-verify
```

Saved comparison path preserved:

```text
run / ingest -> collect -> compare --save -> validate-comparison -> package-comparison -> package-verify
```

## Implemented Behavior

- Version metadata is advanced to `0.1.6` in `pyproject.toml` and
  `lab_sidecar/__init__.py`.
- `CHANGELOG.md` includes a `0.1.6` entry dated 2026-06-25.
- README onboarding now points first users to a focused release-wheel
  quickstart and recipe gallery.
- `docs/first-user-quickstart.md` documents a GitHub release wheel install,
  virtual environment setup, source-archive examples, `labsidecar --help`,
  `doctor`, demo run, artifact generation, validation, package creation, and
  package verification.
- `docs/recipes.md` documents three short recipes: local training run,
  explicit messy CSV/JSON ingest, and saved comparison packaging.
- `docs/public-alpha-quickstart.md` now treats release-wheel install as the
  public-alpha default and keeps editable install as development-only.
- `docs/first-user-quickstart.md` uses v0.1.6 as the concrete downloadable
  example for release-finalized docs; the separate v0.1.5 network smoke below
  remains historical evidence from the already-published release asset.
- Release-facing docs record that v0.1.6 does not create a tag, GitHub release,
  or PyPI publish in this implementation slice.
- Release asset URL handling has no-network unit coverage for accepting
  HTTP(S) `.whl` URLs and rejecting non-wheel HTTP(S) URLs.

## Known Limits

- The v0.1.6 implementation slice itself did not create a GitHub release. The
  first-user quickstart is prepared for the v0.1.6 release-finalized artifact.
- `scripts/release_asset_smoke.py` verifies wheels, not sdists. The sdist is
  built and metadata-checked by `scripts/release_check.py`.
- The real GitHub release asset smoke is manual and networked. It is not part
  of normal tests.
- Recipe fixtures are representative local examples, not broad format promises.
- Packages and traceability may contain local paths or command previews that
  humans should review before sharing.
