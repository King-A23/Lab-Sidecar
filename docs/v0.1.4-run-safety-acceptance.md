# v0.1.4 Run Safety Path Acceptance

Date: 2026-06-25

## Scope

This acceptance record covers the v0.1.4 additive CLI argv/non-shell run path.
It does not accept any Web UI, FastAPI/HTTP service, hosted behavior, remote
runner, OS sandboxing, default AI analysis, statistical conclusion, tag,
GitHub release, or PyPI publish.

## Implemented Behavior

- Legacy shell mode remains available as:

  ```bash
  labsidecar run "<command>"
  ```

  It continues to execute with `subprocess.Popen(command, shell=True)`.

- New opt-in argv mode is available as:

  ```bash
  labsidecar run --no-shell -- <program> <arg> ...
  ```

  It executes with `subprocess.Popen(argv, shell=False)`.

- Foreground and background argv runs preserve the argv list as structured
  metadata instead of flattening it into a shell command.

- Run task manifests now include additive run metadata:
  `run_mode`, `argv`, and `safe_profile`.

- Run tasks now write `reproduce/run.json` with structured run metadata.
  `reproduce/command.txt` remains a human-readable preview.

- Task-local traceability records `run_mode`, `argv`, `safe_profile`, and
  `run_spec_path` without embedding full logs or artifact bodies.

- Single-task packages include `reproduce/run.json` when present and retain the
  existing default omission contract for full logs, raw sources, SQLite, worker
  audit files, sandbox files, and unrelated workspace files.

## Known Limits

- argv mode is non-shell execution, not OS sandboxing.
- argv mode does not validate that a command is safe; it only avoids shell
  parsing, expansion, and chaining for that run.
- Legacy shell mode intentionally remains the default for compatibility.
- Safe profiles are not implemented in this slice; `safe_profile` remains
  reserved metadata.
- MCP/V2 safety gates remain separate from manual CLI run behavior.

## Not Done

- No MCP/V2 schema or product-surface expansion.
- No safe-profile policy engine.
- No remote runner, hosted service, Web UI, FastAPI app, cloud sync, or general
  multi-agent product framework.
- No tag, GitHub release, or PyPI publish.

Final review follow-up:

- Fixed argv parser behavior so child flags such as `--name` remain in argv
  instead of being parsed as Lab-Sidecar options once the argv program token
  starts.
- Added effective shell-mode handling for legacy run manifests that predate
  `run_mode`.
- Kept exact argv in task-local `manifest.json` and `reproduce/run.json`, while
  package summaries use `argv_count` and `run_spec_path` instead of embedding
  the full argv list.
