# v0.1.3 Real Sample Robustness Acceptance

Date: 2026-06-24

## Goal

Record the v0.1.3 real sample robustness pass for small representative local
fixtures, without expanding Lab-Sidecar beyond the CLI-first, file-first,
local-first artifact workflow.

Primary path:

```text
run / ingest -> collect -> figures -> report -> slides -> validate -> package -> package-verify
```

Comparison path:

```text
run / ingest -> collect -> compare --save -> validate-comparison -> package-comparison -> package-verify
```

## Scope Confirmed

This pass added checked-in real-ish fixtures and bounded diagnostics for:

- messy CSV source selection with explicit `collect --config`;
- JSON benchmark collection with explicit `collect --config`;
- deterministic figure defaults on sparse and multi-seed samples;
- deterministic report and static PPTX generation on real-ish samples;
- validation, packaging, package verification, and saved comparison regression
  coverage.

It did not add new product surfaces or broader format support.

## Known Limitations

- Fixtures are representative and small; they are not a claim that arbitrary
  messy local data is supported.
- Explicit `collect --config` remains the recommended path for nested or messy
  directories.
- Mixed units are diagnosed but not converted.
- JSONL, TensorBoard, full MLflow tracking-store parsing, broad recursive
  workspace scanning by default, interactive charts, animation, and video
  remain out of scope.
- Comparisons remain bounded, final-row, shared-finite-numeric, and
  descriptive only.
