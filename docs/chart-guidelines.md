# Lab-Sidecar V1 图表生成规范

本规范只覆盖 V1 首批静态图表和表格，目标是“课程实验报告和 PPT 可直接使用”，不追求论文级自动排版。

## 通用默认值

- 默认导出 `PNG + SVG` 两份。
- PNG 默认白底，长边不低于 1600 像素，建议 200 DPI。
- SVG 应保留矢量文本和线条，不把整张图栅格化。
- 标题、坐标轴、图例必须可读；无标题或无坐标轴标签视为不合格。
- 不使用 3D 图、双 Y 轴、彩虹渐变、装饰性阴影。
- 默认调色板应色盲友好，并在同一任务内保持一致。

推荐调色板顺序：

- `#4E79A7`
- `#F28E2B`
- `#59A14F`
- `#E15759`
- `#76B7B2`
- `#9C755F`

额外规则：

- 基线模型优先使用中性灰或深蓝。
- 最佳结果允许用更深的强调色，但不能只靠颜色表达含义。
- 如果中文标题或标签无法正确渲染，应先更换到支持中文的字体，再导出；不能带着乱码继续输出。

## 1. 单实验训练曲线

**适用场景**

单个实验随 `epoch` 或 `step` 变化的指标走势，例如 `train_loss`、`val_loss`、`val_accuracy`。

**输入数据格式**

- 一个 CSV 文件，或一个可转换为表格的 JSON。
- 每一行表示一个时间步。

**必需字段**

- `epoch` 或 `step` 中至少一个
- 至少一个数值型指标列，例如 `train_loss`、`val_accuracy`

**可选字段**

- `split`
- `run_name`
- `seed`
- `best_epoch`

**默认标题规则**

- 优先：`<task_name>: <metric> over <Epoch|Step>`
- 无 `task_name` 时：`<metric> over <Epoch|Step>`

**坐标轴规则**

- X 轴必须按 `epoch` 或 `step` 升序。
- Y 轴名称必须使用指标名或映射后的友好名称。
- `accuracy`、`precision`、`recall`、`f1` 默认限制在 `0-1` 或 `0-100%` 的一致范围内，不混用。

**图例规则**

- 只有一条线时可以省略图例。
- 有多条线时，图例名称优先使用 `train`、`val` 等简短标签。

**颜色规则**

- `train` 默认蓝色。
- `val` 默认橙色。
- 同一张图超过 4 条线时应拒绝自动生成，除非显式指定筛选。

**PNG / SVG 导出要求**

- PNG 与 SVG 文件名保持同名不同后缀。
- SVG 中的线条和文本必须可缩放。

**什么时候应该拒绝生成**

- 数据点少于 2 个。
- X 轴字段不存在或无法排序。
- Y 轴列不是数值型。
- 单图默认需要展示的线条超过 4 条。

**合格 / 不合格示例说明**

- 合格：`epoch=1..20`、`train_loss` 与 `val_loss` 两条线，标题明确，坐标轴完整。
- 不合格：把 `loss`、`accuracy`、`learning_rate` 同时画在同一 Y 轴且没有单位说明。

## 2. 多实验对比曲线

**适用场景**

比较多个模型、方法或配置在相同 `epoch` / `step` 轴上的表现。

**输入数据格式**

- 多个结构一致的 CSV 文件，或一个包含 `experiment` 列的合并表。

**必需字段**

- `epoch` 或 `step`
- 一个数值型指标列
- `experiment`、`model` 或可由文件名映射得到的实验标签

**可选字段**

- `seed`
- `split`
- `dataset`

**默认标题规则**

- `<metric> comparison`
- 若数据集已知：`<dataset> <metric> comparison`

**坐标轴规则**

- 所有曲线必须共享同一 X 轴语义和同一 Y 轴单位。
- 如果一部分数据是 `epoch`，另一部分是 `step`，应拒绝自动合图。

**图例规则**

- 图例必须使用简短实验名，例如 `baseline`、`model_a`。
- 默认最多展示 8 条曲线；超过 8 条应拒绝并提示先筛选。

**颜色规则**

- `baseline` 优先灰色或深蓝。
- 其余实验按调色板顺序分配。
- 同一实验在不同图中的颜色应保持一致。

**PNG / SVG 导出要求**

- PNG 至少满足 16:9 展示清晰度。
- SVG 不应把不同曲线合并成无法单独辨识的路径组。

**什么时候应该拒绝生成**

- 指标单位混用。
- 实验标签缺失且无法从文件名可靠推断。
- 曲线数量超过 8 条。
- 数据跨度过大且没有显式要求对数坐标。

**合格 / 不合格示例说明**

- 合格：`baseline`、`model_a`、`model_b` 三条 `val_accuracy` 曲线，图例清楚。
- 不合格：12 条不同 seed 曲线直接堆在一张图上，没有聚合也没有筛选。

## 3. 最终指标柱状图

**适用场景**

比较不同实验、模型或数据集上的最终指标，例如最终准确率、F1、延迟或吞吐。

**输入数据格式**

- 一行一个实验结果的 CSV 或 JSON 表。

**必需字段**

- 一个类别字段，例如 `experiment`、`model`、`algorithm`
- 一个数值型最终指标字段，例如 `final_accuracy`

**可选字段**

- `std`
- `ci95`
- `dataset`
- `rank`

**默认标题规则**

- `Final <metric> by <group>`

**坐标轴规则**

- X 轴为类别。
- Y 轴必须写明指标与单位，例如 `Accuracy (%)`、`Runtime (ms)`。
- 单位不同的数据不能放在同一张柱状图中。

**图例规则**

- 只有单指标单分组时通常不需要图例。
- 分组柱状图才显示图例。

**颜色规则**

- 基线柱使用中性灰。
- 最优值可加深颜色，但不要仅靠颜色表达“最佳”，可同时加粗数值标注或排序。

**PNG / SVG 导出要求**

- 类别名称必须完整显示，不允许被截断。
- 当类别较多时，允许自动旋转 X 轴标签，但仍须保持可读。

**什么时候应该拒绝生成**

- 类别超过 12 个且没有显式筛选。
- 数值列为空或非数值。
- 同一图中混入不同单位。

**合格 / 不合格示例说明**

- 合格：4 个模型的 `Accuracy (%)` 柱状图，基线和最优值一眼可辨。
- 不合格：把 `accuracy` 和 `runtime_ms` 画成同一组柱，标题只写 `Results`。

## 4. 多 seed 均值 / 方差图

**适用场景**

展示同一实验或多种方法在多个 seed 下的平均表现和波动范围。

**输入数据格式**

- 明细表，每行至少包含 `seed` 和一个数值指标。

**必需字段**

- `seed`
- `epoch` 或 `step`，如果是曲线型均值图
- 数值型指标列
- `experiment` 或 `model`，如果要比较多个方法

**可选字段**

- `std`
- `variance`
- `ci95`
- `dataset`

**默认标题规则**

- `<metric> mean ± variance`
- 若有模型分组：`<metric> mean ± variance by experiment`

**坐标轴规则**

- 曲线型图默认画均值线，并以阴影或误差条表达波动。
- 至少 2 个 seed 才允许生成；建议 3 个及以上。

**图例规则**

- 每个实验一条均值线。
- 阴影不进入单独图例项，避免图例膨胀。

**颜色规则**

- 均值线使用正常调色板。
- 方差阴影使用同色低透明度。

**PNG / SVG 导出要求**

- 阴影区域在 PNG 和 SVG 中都必须可辨识。
- 导出时不要把误差条或阴影裁切出边界。

**什么时候应该拒绝生成**

- 少于 2 个 seed。
- 缺少可聚合字段。
- 方差来源不清楚，例如把不同数据集混在一起聚合。

**合格 / 不合格示例说明**

- 合格：3 个 seed 的 `val_accuracy` 均值曲线，带标准差阴影。
- 不合格：只有 1 个 seed 却仍然标成“mean ± std”。

## 5. 消融实验结果表

**适用场景**

比较基线与若干消融变体的最终结果，适合课程报告中的结果表和 PPT 截图。

**输入数据格式**

- CSV 或 JSON 表，每行一个变体。

**必需字段**

- `variant`
- 至少一个数值指标列，例如 `accuracy`、`f1`、`latency_ms`

**可选字段**

- `delta_vs_baseline`
- `notes`
- `rank`
- `dataset`

**默认标题规则**

- `Ablation Results`
- 若任务名已知：`<task_name> Ablation Results`

**坐标轴规则**

- 不适用。
- 单位必须写进列表头，例如 `Latency (ms)`。

**图例规则**

- 不使用图例。

**颜色规则**

- 基线行浅灰底。
- 最佳值允许加粗。
- 只有在 `delta_vs_baseline` 明确存在时，才允许用绿色/红色标示提升或下降。

**PNG / SVG 导出要求**

- 主要交付物应同时包括 `table.csv` 和 `table.md`。
- 若导出图片，PNG 必须清晰可投屏；SVG 需保证文本可选中。

**什么时候应该拒绝生成**

- 变体少于 2 个。
- 指标列过多，导致单张表无法清晰阅读；默认上限 6 列。
- 行标题或注释过长，超出默认版面且无法自动换行。

**合格 / 不合格示例说明**

- 合格：`baseline`、`-dropout`、`-augmentation` 三行，列头带单位，最佳值清楚。
- 不合格：10 多个变体和 12 个指标硬塞到一张小图里，表头没有任何单位说明。
